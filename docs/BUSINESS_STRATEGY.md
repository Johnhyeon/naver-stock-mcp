## 유료 MCP의 결제 구조

MCP 자체에는 결제 기능이 없습니다. 돈을 받는 건 **MCP 바깥에서** 일어납니다.

### 구조

```
1. 사용자가 웹사이트에서 결제 (월 구독)
2. API 키 발급받음
3. MCP 설정에 API 키 입력
4. MCP 서버가 매 요청마다 API 키를 백엔드에 검증
5. 유효하면 데이터 반환, 아니면 거부
```

### 실제 예시: Alpha Vantage

```
1. alphavantage.co에서 회원가입 + 결제 ($50/월)
2. API 키 받음: "ABCD1234EFGH"
3. MCP 설정:
   {
     "mcpServers": {
       "alpha-vantage": {
         "command": "alpha-vantage-mcp",
         "env": {
           "ALPHA_VANTAGE_API_KEY": "ABCD1234EFGH"
         }
       }
     }
   }
4. MCP 서버가 데이터 요청 시 이 키를 Alpha Vantage API에 전달
5. Alpha Vantage가 키 확인 → 유료 사용자면 응답, 아니면 거부
```

**MCP 서버는 중간 다리일 뿐, 실제 데이터와 결제는 백엔드 API에서 처리**

### 우리 케이스

```
무료: 네이버 증권 공개 데이터 (누구나 접근 가능)
     → API 키 불필요 → MCP 무료

유료: stock_tracker의 Smart Scanner, 촉매 감지, AI 내러티브 등
     → 당신 서버에서만 돌아가는 분석 결과
     → 이 서버 접근 권한 = API 키 = 유료
```

네이버 데이터는 공개라 돈을 못 받지만, **당신이 그 데이터로 만든 분석 결과**는 당신 서버에만 있으니 돈을 받을 수 있는 구조입니다.

**정확히 짚으셨습니다.** 근본 원인은 MCP 구조 자체가 아니라 **"DB가 없다"는 것**입니다.

## HTS vs 우리 MCP 비교

```
HTS (증권사 시스템):
  사용자 질문
    ↓
  자체 DB에 실시간으로 쌓아둔 전체 종목 데이터
    ↓
  SELECT * FROM stocks WHERE per < 10 AND foreign_net_buy > 0  ← 0.01초
    ↓
  결과 즉시

우리 MCP:
  사용자 질문
    ↓
  종목 하나 조회 (HTTP 요청)
    ↓
  다음 종목 조회 (HTTP 요청)
    ↓ ...
  100번 반복                                                     ← 10~30초
    ↓
  결과
```

HTS는 **데이터가 이미 메모리/DB에 있어서** 조건 검색이 즉시 되고, 우리는 **매번 네이버에 물어봐야** 하니 느립니다.

## 내가 제시한 방식 (get_multi_financials)으로 해결되나?

**부분적으로 해결됩니다.** 구체적으로:

### 개선되는 부분
```
Before: get_financial × 100번 순차 호출
  → Claude 응답 지연 + 네이버 요청 시간 누적
  → 예상 30~60초

After: get_multi_financials(100개) 1번 호출
  → 내부에서 asyncio.gather로 병렬
  → 예상 2~3초
```

**약 10~20배 빨라집니다.** 하지만 여전히 HTS의 "0.01초"에는 못 미칩니다.

### 더 빠르게 하려면? (근본 해결책)

**우리도 DB를 갖자.** 즉:

```
1. 서버 시작 시 (또는 매일 아침 9시)
   → 네이버에서 전체 종목 데이터 한 번에 수집
   → 로컬 SQLite/Redis에 저장

2. 사용자 질문 시
   → 로컬 DB SELECT ← HTS처럼 즉시 응답

3. 실시간 데이터(현재가 등)만 네이버에 질의
```

리서치에서 본 **MaverickMCP**가 이 방식입니다 (Redis + smart cache merge).

## 네이버 크롤링 법적 상태

### 한국 법 기준 (정보통신망법/저작권법)

**"공개된 데이터의 개인적 이용"은 일반적으로 허용**됩니다:
- 종목 시세/재무지표는 **공공재 성격** (KRX가 원천 제공)
- 개인 투자자가 자신의 투자 판단을 위해 보는 건 문제 없음
- 네이버는 단순 **게시자** (원천 데이터 소유 아님)

**허용되는 범위:**
- 개인이 본인 PC에서 본인을 위해 조회 → OK
- 분산 부하 (각 사용자의 IP에서) → OK
- 비정상적 대량 수집이 아닌 정상 수준 → OK

**문제가 되는 경우:**
- 수집한 데이터를 **재판매/재배포** → 위반 가능성
- 과도한 부하로 서비스에 지장 → 업무방해죄 가능
- 네이버 이용약관 위반

## 우리 상황 분석

### 현재 (무료 + 로컬 수집) → 안전 ✓

```
사용자 A의 PC → 네이버에 접속 → 자기 데이터 조회
사용자 B의 PC → 네이버에 접속 → 자기 데이터 조회
...
```

**각 개인이 직접 네이버를 조회하는 것과 동일**. 네이버 입장에서는 수만 명이 브라우저로 접속하는 것과 MCP로 접속하는 것의 차이가 없음.

### 유료 판매 시 → 그레이존 ⚠️

**무엇을 파느냐에 따라 리스크가 달라짐:**

#### 케이스 A: "도구(소프트웨어)를 판매" — 저위험
```
판매 물품: MCP 서버 코드 + 설치 가이드 + 프롬프트 템플릿
데이터: 여전히 사용자가 본인 PC에서 직접 수집
```

**"엑셀 매크로나 트레이딩 봇을 판매하는 것"**과 동일한 구조. 합법성 높음.

#### 케이스 B: "서비스를 판매" — 그레이존
```
판매 물품: 월 구독 → "매일 아침 스크리닝 결과 이메일"
데이터 흐름: 당신 서버 → 네이버 수집 → 가공 → 고객에게 배포
```

**데이터 재배포**에 해당, 리스크 급증.

#### 케이스 C: "분석 결과 판매" — 안전
```
판매 물품: 월 구독 → "이번 주 AI가 찾은 관심 종목 분석 리포트"
데이터 흐름: 수집 → AI 분석 → 분석 결과(의견)만 판매
```

**데이터가 아니라 "분석/의견"을 파는 구조**. 단 한국에서는 "불특정 다수에게 투자 의견 제공"은 유사투자자문업 신고 필요.

**장점:**
- 제작 시간 1~2주
- 가격 부담 낮음 (9,900~19,900원)
- 즉시 판매 가능
- 업데이트 쉬움

**구성안 (60~100페이지):**
```

Chapter 5. 상황별 프롬프트 템플릿 50개

Chapter 7. 실전 시나리오 10선
Chapter 8. 자주 실수하는 패턴 + 해결법
Appendix. 전체 도구 레퍼런스
```

**장점:**
- 가격 훨씬 높음 (49,000~99,000원)
- 높은 완성도 인식
- 커뮤니티 기반 만들기 좋음

**단점:**
- 제작 시간 2~3개월
- 영상 품질 이슈
- 업데이트 어려움


## 3. 리포트 서비스 — 펜딩이 맞습니다

### 스팸 이슈 외에 큰 문제

**법적 리스크: 유사투자자문업 등록**

```
"불특정 다수에게 투자 조언/의견을 정기적으로 제공" = 유사투자자문업
→ 금감원 등록 필수
→ 미등록 시 과태료 최대 1억원
```

"매일 종목 추천 이메일"은 명확히 여기 해당함.

### 우회 방법

**대안 1: 순수 "교육 콘텐츠" 뉴스레터**
```
❌ "오늘의 추천 종목: 삼성전자 매수"
✅ "오늘의 시장 해석: 외국인 수급 동향 분석 방법"
```

**대안 2: 유료 회원 전용 (클로즈드)**
```
불특정 다수 → 특정 유료 회원
"투자자문"의 '불특정 다수' 요건 회피 가능성
```

**대안 3: 아예 "데이터 요약"만 제공**
```
"이번 주 거래량 상위 10개 종목 리스트 (단순 통계)"
분석/의견 없이 데이터만 → 완전 안전
```

### 결론

**리포트 서비스는 Phase 3~4로 미루세요.** 대신 이메일 리스트는 **교육 뉴스레터** 용도로 쓰면 됨:

```
매주 목요일 발송:
- 이번 주 MCP 업데이트 소식
- 새로 추가된 프롬프트 템플릿
- 시장 분석 기법 1개 (교육용)
- 사용자 질문 QnA
```

이런 뉴스레터는 스팸 아니고 가치 제공이라 환영받고, 유료 상품 출시 시 전환율이 매우 높습니다.

## 파생 상품 후보 7개

### 1. Excel 플러그인 ⭐⭐⭐⭐⭐ (가장 유망)

```
=KSTOCK_PRICE("005930")           → 현재가
=KSTOCK_PER("005930")              → PER
=KSTOCK_FOREIGN_NET("005930", 20) → 외국인 20일 순매수
=KSTOCK_SCREEN("per<10 roe>15")   → 조건 충족 종목 배열
```

- **타겟**: 엑셀로 주식 관리하는 40~60대 직장인 (거대 시장)
- **Claude 불필요**: 100% 독립
- **수익 모델**: 월 9,900원 ~ 19,900원, 또는 일회성 49,000원
- **개발 난이도**: 중

**왜 유망:**
- "AI는 무서운데 엑셀은 편하다" 사용자 많음
- 엑셀 의존도 높은 중장년층이 핵심층
- 경쟁 제품 거의 없음 (데이터가이드 정도)

### 2. 브라우저 확장 프로그램 ⭐⭐⭐⭐

```
사용자가 네이버 증권 종목 페이지 접속
   ↓
확장 프로그램이 자동으로 분석 레이어 추가
   ├─ "AI 분석" 버튼 (사용자 Gemini/OpenAI 키)
   ├─ 시그널 스캐너 결과
   ├─ 외국인 수급 차트 추가
   └─ 유사 종목 추천
```

- **타겟**: 네이버 증권 사용자 (압도적 다수)
- **Claude 불필요**: 사용자 선택 (Claude/GPT/Gemini)
- **수익 모델**: Freemium, Pro 월 5,900원
- **개발 난이도**: 중

### 3. 데스크톱 앱 ⭐⭐⭐

```
   └─ 로컬 작동
```

- **타겟**: "AI 어려운데 도구는 원하는" 중급 투자자
- **Claude 불필요**: 선택
- **수익 모델**: 일회성 79,000원 or 월 14,900원
- **장점**: stock_tracker React 프론트 **거의 재활용** 가능

### 4. 텔레그램 봇 ⭐⭐⭐⭐

```
/scan 반도체 외국인순매수  → 스크리닝 결과
/alert 005930 저항선돌파    → 조건부 알림
/daily                      → 매일 시장 요약
```

- **타겟**: 모바일 우선 사용자, 젊은 투자자
- **Claude 불필요**: Gemini or 로컬 분석
- **수익 모델**: Free → Pro 월 5,900원

### 5. Python SDK (개발자용) ⭐⭐

```python
from kstock import Screener, Analyzer

s = Screener()
candidates = s.scan(per_max=10, pbr_max=1, foreign_buying_days=5)

a = Analyzer()
for stock in candidates:
    print(a.winstein_stage(stock))
```

- **타겟**: 퀀트 개발자
- **수익 모델**: Free → Pro 월 29,900원
- **개발 난이도**: 낮음 (stock_tracker 서비스 래핑)
